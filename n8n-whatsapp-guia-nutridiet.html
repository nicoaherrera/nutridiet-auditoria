🗺️ 0. Cómo funciona todo junto

Son 4 piezas que se comunican entre sí. Una vez conectadas, el bot funciona solo sin tocar nada más.

📱
WhatsApp
El cliente escribe. Meta lo recibe y lo manda a n8n.
🔵
Meta Cloud API
Recibe y envía mensajes. El puente entre WhatsApp y tu bot.
🩷
n8n
Cerebro del bot. Lee el mensaje, decide qué responder, lo envía.
🟢
Google Sheets
Memoria. Guarda en qué paso está cada cliente en la conversación.

El viaje de un mensaje

👤 Cliente escribe "Hola" en WhatsApp
↓
🔵 Meta Cloud API recibe el mensaje y lo envía al webhook de n8n
↓
🟢 n8n consulta Google Sheets: ¿en qué estado está este número?
↓
🩷 n8n decide qué responder según el estado y el mensaje
↓
🔵 n8n llama a Meta API para enviar la respuesta
↓
🟢 n8n actualiza el estado del cliente en Google Sheets
↓
📱 Cliente recibe: "¡Hola! Bienvenido a NutriDiet Market..."
⏱️
Todo este proceso tarda menos de 1 segundo. El cliente recibe la respuesta casi instantáneamente.

🔵 Fase 1 — Configurar Meta WhatsApp Cloud API

🔵 Meta for Developers — Tiempo estimado: 30-45 minutos

Paso 1.1 — Crear la app en Meta for Developers

1

Andá a developers.facebook.com → clic en "Mis apps" → "Crear app".

2

En "Tipo de app" elegí "Empresa" (Business). Clic en Siguiente.

3

Nombre de la app: NutriDiet-Bot. Asociala a tu cuenta de Meta Business. Clic en "Crear app".

Paso 1.2 — Agregar WhatsApp a la app

4

En el panel de la app buscá el producto "WhatsApp" y clic en "Configurar".

5

Te va a pedir asociar una cuenta de WhatsApp Business. Si no tenés, creala ahí mismo con el nombre "NutriDiet Market".

6

Meta te da un número de prueba con 5 números de destino para testear. Usalo para probar antes de conectar el número real.

Paso 1.3 — Obtener los datos que necesita n8n

7

En el menú izquierdo andá a WhatsApp → Configuración de la API. Ahí encontrás:

• Token de acceso temporal (empieza con EAA...) — lo necesitás para que n8n pueda enviar mensajes.
• Phone Number ID — el ID numérico del número de WhatsApp.

Copiá y guardá los dos en un bloc de notas.

ℹ️
Token permanente: El token temporal dura 24hs. Para producción necesitás generar un System User Token que no vence. Andá a business.facebook.com → Configuración → Usuarios del sistema → Agregar → Dar permisos de WhatsApp. Esto lo hacés cuando todo esté funcionando.

Paso 1.4 — Agregar tu número de destino para pruebas

8

En WhatsApp → Configuración de la API, buscá "Para" y agregá tu número personal como número de prueba. Meta te manda un código por WhatsApp para verificarlo.

9

Hacé una prueba: clic en "Enviar mensaje". Deberías recibir un "Hello World" en tu WhatsApp. Si llegó, la API está funcionando.

🔗 Fase 4 — Conectar el webhook

🩷 n8n + Meta — Tiempo estimado: 15 minutos

En n8n: crear el workflow principal

1

En n8n, clic en "New Workflow" → nombralo NutriDiet — Bot Principal.

2

Agregá el primer nodo: buscá "Webhook" → arrastralo al canvas.

3

Configurá el nodo Webhook así:
• HTTP Method: POST
• Path: whatsapp
• Response Mode: Immediately (importante: Meta necesita respuesta inmediata)
• Response Code: 200

La URL del webhook queda: https://nutridiet-n8n.up.railway.app/webhook/whatsapp

En Meta: registrar el webhook

4

En Meta for Developers → tu app → WhatsApp → Configuración → buscá la sección "Webhooks" → clic en "Configurar webhooks".

5

Completá:
• URL de devolución de llamada: https://nutridiet-n8n.up.railway.app/webhook/whatsapp
• Token de verificación: Inventá una palabra secreta, ej: nutridiet2026 (la vas a necesitar también en n8n)
• Clic en "Verificar y guardar".

6

En la sección de campos del webhook, activá el toggle de messages. Guardá.

✅
Si Meta pudo verificar el webhook significa que n8n está escuchando correctamente. Ya hay conexión entre los dos.

🩷 Fase 5 — Construir el flujo principal en n8n

🩷 n8n Workflow — Tiempo estimado: 60-90 minutos

Este es el corazón del bot. Cada vez que llega un mensaje, este workflow decide qué responder. Los textos exactos de cada mensaje ya están en el archivo bot-flujos-textos-nutridiet.html — acá solo definimos la lógica de cuándo enviar cada uno.

Nodo 1 — Webhook (ya creado en Fase 4)

WebhookRecibir mensaje de Meta
Ya configurado. Recibe el JSON de Meta con toda la info del mensaje.

Nodo 2 — Extraer datos del mensaje

SetExtraer: teléfono, mensaje, tipo

Creá un nodo Set y definí estas variables:

  • telefono = {{ $json.entry[0].changes[0].value.messages[0].from }}
  • mensaje = {{ $json.entry[0].changes[0].value.messages[0].text.body }}
  • tipo_msg = {{ $json.entry[0].changes[0].value.messages[0].type }}
  • phone_number_id = {{ $json.entry[0].changes[0].value.metadata.phone_number_id }}

Nodo 3 — Verificar si es un mensaje válido

IF¿Es un mensaje de texto?
  • Condición: tipo_msg igual a text
  • Si es true: seguir al nodo 4
  • Si es false: puede ser imagen (comprobante de pago) → flujo de pago

Nodo 4 — Buscar estado del usuario en Google Sheets

Google SheetsBuscar estado del usuario
  • Operation: Lookup
  • Spreadsheet: NutriDiet Bot — Estados (el que creaste)
  • Sheet: Sheet1
  • Lookup Column: TELEFONO
  • Lookup Value: {{ $json.telefono }}
  • Si no encuentra resultado → el estado es nuevo

Nodo 5 — Switch por estado

Switch¿En qué estado está el usuario?

Creá una salida para cada estado. El valor a evaluar es el campo ESTADO de Sheets:

  • Salida 1: ESTADO = nuevo o vacío → Flujo Bienvenida
  • Salida 2: ESTADO = menu_mostrado → Procesar opción del menú
  • Salida 3: ESTADO = esperando_productos → Flujo Pedido
  • Salida 4: ESTADO = esperando_confirmacion → Flujo Confirmación
  • Salida 5: ESTADO = esperando_tipo_entrega → Flujo Entrega
  • Salida 6: ESTADO = esperando_direccion → Capturar dirección
  • Salida 7: ESTADO = esperando_nombre → Capturar nombre
  • Salida 8: ESTADO = esperando_pago → Capturar forma de pago

Nodo 6 — Enviar mensaje (se repite para cada respuesta)

HTTP RequestEnviar mensaje por Meta API

Este nodo lo vas a reusar para cada mensaje que el bot envía. Configuración base:

  • Method: POST
  • URL: https://graph.facebook.com/v19.0/{{ $json.phone_number_id }}/messages
  • Authentication: Header Auth → Authorization: Bearer TU_TOKEN_META
  • Body Type: JSON

JSON del cuerpo para enviar un mensaje de texto

COPIAR EN HTTP REQUEST — Body JSON
{ "messaging_product": "whatsapp", "recipient_type": "individual", "to": "{{ $('Set').item.json.telefono }}", "type": "text", "text": { "preview_url": false, "body": "¡Hola! 👋 Bienvenido/a a *NutriDiet Market* 🌿\n\n🛒 *1* — Ver catálogo\n📦 *2* — Hacer un pedido\n🚚 *3* — Consultar envíos\n📍 *4* — Horarios\n💬 *5* — Hablar con una persona" } }
💡
Reemplazá el body con el texto de cada mensaje del archivo bot-flujos-textos-nutridiet.html. Los saltos de línea van como \n en el JSON. El negrita de WhatsApp se hace con *texto*.

Nodo 7 — Actualizar estado en Sheets

Google SheetsActualizar estado del usuario

Después de cada respuesta, actualizás el estado:

  • Operation: Update (si existe) o Append (si es nuevo)
  • TELEFONO: {{ $json.telefono }}
  • ESTADO: el nuevo estado (ej: menu_mostrado)
  • DATOS: JSON con los datos que vas acumulando del pedido
  • ACTUALIZADO: {{ $now }}

Cómo acumular datos del pedido en Sheets

El campo DATOS es un JSON que va creciendo a medida que el cliente avanza. En cada paso, n8n lee el JSON anterior, le agrega el nuevo dato y lo guarda.

FUNCIÓN EN N8N — Acumular datos del pedido
// Nodo Function para combinar datos anteriores con el nuevo dato const datosActuales = JSON.parse($input.item.json.DATOS || '{}'); // Agregar el nuevo dato según el estado datosActuales.productos = $input.item.json.mensaje; // Devolver el JSON actualizado como string para Sheets return { datos_actualizados: JSON.stringify(datosActuales) };

Notificar al empleado cuando llega un comprobante

HTTP RequestNotificar al empleado por WhatsApp

Cuando el cliente manda el comprobante (tipo image), n8n envía un mensaje al número del empleado con el resumen del pedido. Usá el mismo nodo HTTP Request pero con el número del empleado como destino.

MENSAJE DE NOTIFICACIÓN AL EMPLEADO
⚠️ *NUEVO PEDIDO CON PAGO* ⚠️ 📞 Cliente: {{ $json.telefono }} 👤 Nombre: {{ datos.nombre }} 🛒 Productos: {{ datos.productos }} 📍 Dirección: {{ datos.direccion }} 💳 Pago: {{ datos.metodo_pago }} El cliente acaba de enviar el comprobante. Verificar y confirmar.

🧪 Fase 8 — Probar que funciona

🧪 Testing — Tiempo estimado: 30 minutos

Checklist de pruebas

1

Escribí "Hola" desde tu WhatsApp personal al número del bot. Deberías recibir el menú de bienvenida en menos de 2 segundos.

2

Respondé "1" → deberías recibir el mensaje del catálogo.

3

Iniciá desde cero: escribí "2" → completá el flujo de pedido hasta el final (productos → dirección → nombre → pago). Verificá en Google Sheets que se van actualizando los estados.

4

Mandá una imagen cualquiera cuando el estado sea esperando_pago → verificá que el empleado recibe la notificación.

5

Escribí algo sin sentido dos veces seguidas → verificá que al segundo intento el bot dice que te pasa con una persona.

6

Probá desde un segundo número distinto al mismo tiempo para verificar que los estados de ambos se guardan por separado en Sheets.

🎉
Si todas las pruebas pasan, el bot está listo para recibir clientes reales. El siguiente paso es conectar el número real del negocio siguiendo el proceso de Meta para migrar un número a la API.
